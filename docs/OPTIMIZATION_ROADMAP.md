# 历史生存模拟器 - 优化升级路径规划

> 本文档为历史生存模拟器游戏提供全面的优化、升级、功能扩展规划路线图。

## 一、游戏现状总结

### 1.1 核心框架
- **框架与剧情分离**：采用 JSON 格式剧本，完全解耦游戏逻辑与内容
- **多卷章节解锁机制**：强制从第一卷开始，通关解锁下一卷
- **多身份系统**：每个章节包含多个可选身份
- **属性系统**：生命值、求救次数、负面状态等可配置属性
- **随机选项池**：支持从选项池随机抽取题目选项
- **限时答题机制**：超时扣血，考验历史知识掌握程度
- **特殊事件系统**：支持条件触发的一次性事件
- **成绩评定与复盘**：多维度评分、薄弱知识点分析

### 1.2 已实现功能
| 模块 | 功能点 |
|------|--------|
| 剧本管理 | 内置剧本、自定义剧本上传/删除/多剧本管理 |
| 卷系统 | 第一卷自动解锁、通关解锁下一卷、解锁状态持久化 |
| 身份系统 | 身份卡片选择、单身份自动跳过 |
| 关卡系统 | 滑块选择关卡数量、随机选项池抽取 |
| 属性系统 | 通用属性更新、属性上限/下限控制 |
| 倒计时 | 多级配置覆盖、超时扣血、弹窗冻结 |
| 求救机制 | 正确答案提示、消耗求救机会保命 |
| 事件系统 | beforeLevelStart/afterOptionSelect 触发时机 |
| 成绩评定 | 多维度评分、等级评定、薄弱知识点分析 |
| 复盘系统 | 完整闯关记录、选项对比 |
| 存档系统 | 本地存储、存档保存/加载/清除 |

### 1.3 技术栈
- **前端**：原生 HTML/CSS/JavaScript，无框架依赖
- **存储**：localStorage 本地持久化
- **样式**：CSS Variables、Flexbox、Grid、古风主题
- **部署**：纯静态文件，可直接部署至任意 Web 服务器

## 二、优化升级方向

> ⚠️ **重要更新**：经过深度核查，部分功能已被删除或后移。详见 `FEASIBILITY_REVIEW.md`。

### 2.1 可玩性增强

#### 2.1.1 成就系统 ⭐
- **描述**：记录玩家特殊成就，增加长期粘性
- **功能点**（简化版，5-6个核心成就）：
  - 初出茅庐（首次通关任意卷）
  - 连胜达人（连续答对5题）
  - 极限生存（生命值≤10通关）
  - 知识渊博（答对所有题通关）
  - 全家福（体验所有身份）
- **删除内容**：过于复杂的成就分级
- **建议**：P1 优先实施

#### 2.1.2 道具系统 ⭐（简化版）
- **描述**：增加策略选择，但保持简洁
- **保留道具**（仅3种）：
  - 延时沙漏（+5秒答题时间）
  - 生命药水（+30HP）
  - 知识锦囊（排除1个错误选项）
- **删除内容**：护身符、双倍积分券等复杂道具
- **建议**：P1 实施

#### 2.1.3 每日任务 ⭐（简化版）
- **描述**：每日登录和简单任务
- **功能点**：
  - 每日首次登录奖励
  - 每日答对3题奖励
- **删除内容**：复杂任务链、周末双倍等
- **建议**：P1 优先实施

#### 2.1.4 本地排行榜
- **描述**：展示个人最佳成绩
- **功能点**：
  - 各卷最高分
  - 最高正确率
- **建议**：P2 实施

#### 2.1.5 多结局系统（轻量版）
- **描述**：简单结局判定
- **功能点**：
  - 普通结局（通关）
  - 隐藏结局（特定条件触发，如无伤通关）
- **建议**：P2 后移

## 六、风险与挑战

| 风险 | 影响 | 应对策略 |
|------|------|----------|
| 内容创作成本高 | 剧本内容不足 | 开放编辑器、培养创作者社区 |
| 加载性能 | 大量图片影响体验 | 图片懒加载、WebP 格式 |
| 数据同步 | 离线与在线数据冲突 | 本地优先、冲突合并策略 |
| 版权问题 | 历史内容准确性 | 参考权威资料、标注来源 |
| AI 成本 | AI 讲解费用 | 按需调用、缓存结果 |

## 七、总结

历史生存模拟器已经具备坚实的核心框架和完整的功能体系。通过分阶段的优化升级，可以逐步将其打造为：

1. **学习工具**：不只是游戏，更是历史知识学习平台
2. **社交平台**：玩家互动、排行榜、分享传播
3. **创作平台**：UGC 剧本创作与分享
4. **商业产品**：付费剧本、会员体系、广告变现

建议按照 **"核心体验优化 → 学习性增强 → 社交与运营 → 技术智能化"** 的路线稳步推进。
