# 历史生存模拟器 - 优化升级路径规划

> 本文档为历史生存模拟器游戏提供全面的优化、升级、功能扩展规划路线图。

## 一、游戏现状总结

### 1.1 核心框架
- **框架与剧情分离**：采用 JSON 格式剧本，完全解耦游戏逻辑与内容
- **多卷章节解锁机制**：强制从第一卷开始，通关解锁下一卷
- **多身份系统**：每个章节包含多个可选身份
- **属性系统**：生命值、求救次数、负面状态等可配置属性
- **随机选项池**：支持从选项池随机抽取题目选项
- **限时答题机制**：超时扣血，考验历史知识掌握程度
- **特殊事件系统**：支持条件触发的一次性事件
- **成绩评定与复盘**：多维度评分、薄弱知识点分析

### 1.2 已实现功能
| 模块 | 功能点 |
|------|--------|
| 剧本管理 | 内置剧本、自定义剧本上传/删除/多剧本管理 |
| 卷系统 | 第一卷自动解锁、通关解锁下一卷、解锁状态持久化 |
| 身份系统 | 身份卡片选择、单身份自动跳过 |
| 关卡系统 | 滑块选择关卡数量、随机选项池抽取 |
| 属性系统 | 通用属性更新、属性上限/下限控制 |
| 倒计时 | 多级配置覆盖、超时扣血、弹窗冻结 |
| 求救机制 | 正确答案提示、消耗求救机会保命 |
| 事件系统 | beforeLevelStart/afterOptionSelect 触发时机 |
| 成绩评定 | 多维度评分、等级评定、薄弱知识点分析 |
| 复盘系统 | 完整闯关记录、选项对比 |
| 存档系统 | 本地存储、存档保存/加载/清除 |

### 1.3 技术栈
- **前端**：原生 HTML/CSS/JavaScript，无框架依赖
- **存储**：localStorage 本地持久化
- **样式**：CSS Variables、Flexbox、Grid、古风主题
- **部署**：纯静态文件，可直接部署至任意 Web 服务器

## 二、优化升级方向

### 2.1 可玩性增强

#### 2.1.1 成就系统
- **描述**：新增成就系统，记录玩家各种特殊成就
- **功能点**：
  - 首次通关成就（首个卷通关、首个身份通关）
  - 连续正确成就（3连、5连、10连正确答题）
  - 无伤通关（生命值从未低于50通关）
  - 极限通关（生命值≤10通关）
  - 求救达人（使用求救次数≥5通关）
  - 知识渊博（答对所有题通关）
  - 收集成就（体验所有身份）
  - 速通成就（每关平均用时≤3秒）
- **数据结构**：
```javascript
{
  "achievements": [
    {
      "id": "first_pass",
      "name": "初出茅庐",
      "description": "首次通关任意卷",
      "icon": "🎖️",
      "condition": { "type": "volume_passed", "count": 1 }
    }
  ],
  "unlockedAchievements": ["achievement_id_1"],
  "unlockTime": { "achievement_id_1": "2026-03-29" }
}
```

#### 2.1.2 排行榜系统
- **描述**：添加全球/好友排行榜
- **功能点**：
  - 总分排行榜
  - 单卷最高分排行
  - 正确率排行榜
  - 通关速度排行榜
- **实现方式**：
  - 短期：本地排行榜（localStorage）
  - 长期：后端 API + 数据库（可选扩展）

#### 2.1.3 道具系统
- **描述**：新增可购买的道具系统
- **功能点**：
  - 商店：使用游戏内积累的财富值购买道具
  - 道具类型：
    - 延时沙漏（增加单关5秒答题时间）
    - 生命药水（恢复30点生命值）
    - 知识锦囊（显示提示，排除1个错误选项）
    - 护身符（本关免疫必死选项）
    - 双倍积分券（下次通关分数×2）
- **数据结构**：
```javascript
{
  "shop": [
    {
      "id": "time_sand",
      "name": "延时沙漏",
      "description": "本关答题时间+5秒",
      "price": 50,
      "icon": "⏳",
      "effect": { "type": "time_add", "value": 5 }
    }
  ],
  "inventory": { "time_sand": 2, "health_potion": 1 },
  "wealth": 150
}
```

#### 2.1.4 Roguelike 元素
- **描述**：增加随机性和策略选择
- **功能点**：
  - 随机事件卡：在进入关卡前随机触发事件
  - 命运转盘：每5关触发一次命运抉择
  - 隐藏关卡：满足隐藏条件解锁特殊关卡
  - 支线任务：完成额外目标获得奖励

#### 2.1.5 多结局系统
- **描述**：根据玩家属性和选择解锁不同结局
- **功能点**：
  - 普通结局：根据最终评分显示不同评语
  - 隐藏结局：满足特殊条件触发
  - 角色结局：根据身份和属性值显示专属结局
- **数据结构**：

#### 2.6.4 游戏故事脚本生成器 ⭐
- **描述**：降低剧本创作门槛，让普通用户也能创建自己的历史生存剧本
- **定位**：作为剧本编辑器的进阶版本，面向普通用户而非技术用户
- **功能点**：

  **模式一：引导式创建**
  - Step 1：选择时代背景（朝代、具体年份）
  - Step 2：选择主角身份（农民、商人、官员等）
  - Step 3：输入主角姓名和基本信息
  - Step 4：系统生成故事框架，用户填充关键决策点
  - Step 5：系统自动生成选项（正确/错误/致命）
  - Step 6：用户可手动调整每个选项
  - Step 7：生成完整 JSON 剧本

  **模式二：AI 辅助生成**
  - 用户输入故事梗概（100-500字）
  - AI 大模型自动分析并生成完整剧本结构
  - 自动生成关卡、选项、历史知识点
  - 用户可手动优化 AI 生成的脚本

  **模式三：模板修改**
  - 提供多种预设模板（宫廷剧、商战剧、科举剧等）
  - 用户选择模板后修改细节
  - 快速生成自定义剧本

- **数据结构**：
```javascript
{
  "generator": {
    "modes": ["guide", "ai", "template"],
    "guideSteps": [
      { "step": 1, "type": "select", "field": "dynasty", "options": [...] },
      { "step": 2, "type": "select", "field": "identity", "options": [...] },
      { "step": 3, "type": "input", "field": "name" },
      { "step": 4, "type": "textarea", "field": "outline" },
      { "step": 5, "type": "confirm", "field": "generate" }
    ],
    "aiConfig": {
      "apiEndpoint": "/api/generate",
      "promptTemplate": "根据以下故事梗概生成历史生存剧本..."
    }
  }
}
```

- **与剧本编辑器的区别**：
  | 维度 | 剧本编辑器 | 故事脚本生成器 |
  |------|------------|----------------|
  | 目标用户 | 技术用户/创作者 | 普通用户 |
  | 技能要求 | 需了解 JSON 格式 | 无需技术背景 |
  | 操作方式 | 拖拽/表单手动填写 | 引导式/AI 自动生成 |
  | 剧本复杂度 | 可创建完整复杂剧本 | 模板化/简化版剧本 |
  | 定制程度 | 完全自定义 | 半自动生成+手动调整 |

- **简化原则**（保持游戏简洁性）：
  - 生成的剧本默认 5-10 关（可调整）
  - 每个关卡默认 3-4 个选项
  - 提供"一键导出"功能，直接在游戏中使用
  - 不追求完美剧本，追求"可玩性第一"

## 六、风险与挑战

| 风险 | 影响 | 应对策略 |
|------|------|----------|
| 内容创作成本高 | 剧本内容不足 | 开放编辑器、培养创作者社区 |
| 加载性能 | 大量图片影响体验 | 图片懒加载、WebP 格式 |
| 数据同步 | 离线与在线数据冲突 | 本地优先、冲突合并策略 |
| 版权问题 | 历史内容准确性 | 参考权威资料、标注来源 |
| AI 成本 | AI 讲解费用 | 按需调用、缓存结果 |

## 七、总结

历史生存模拟器已经具备坚实的核心框架和完整的功能体系。通过分阶段的优化升级，可以逐步将其打造为：

1. **学习工具**：不只是游戏，更是历史知识学习平台
2. **社交平台**：玩家互动、排行榜、分享传播
3. **创作平台**：UGC 剧本创作与分享
4. **商业产品**：付费剧本、会员体系、广告变现

建议按照 **"核心体验优化 → 学习性增强 → 社交与运营 → 技术智能化"** 的路线稳步推进。
