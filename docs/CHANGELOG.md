# 更新日志

## v2.2.0 (2026-03-30)

### 框架优化

1. **爵位系统按身份差异化**
   - 农户身份：赤贫 → 佃农 → 微农 → 小农 → 中农 → 富农 → 地主 → 乡绅 → 县尉 → 县令 → 太守 → 尚书 → 将军 → 王侯
   - 商户身份：学徒 → 伙计 → 掌柜 → 东家 → 行商 → 大贾 → 豪商 → 皇商 → 省商 → 国商 → 钱庄主 → 工场主 → 商圣 → 首富
   - 学子身份：童生 → 秀才 → 举人 → 贡士 → 进士 → 传胪 → 翰林 → 御史 → 尚书 → 内阁 → 院士 → 大宗师 → 圣人 → 圣王
   - 爵位特权：不同爵位等级解锁不同的游戏加成（生命加成、财富加成、免负面状态、求救次数加成等）

2. **世代传承机制**
   - 卷与身份联动：第一卷选择农户，第二卷只能选择与农户传承相关的身份
   - 属性继承：通关后学识积分可按比例继承到下一卷（默认30%）
   - 剧情连续性：剧本JSON中通过 `inheritFrom` 字段定义卷之间的传承关系
   - 身份过滤：根据上一卷的身份类型自动过滤下一卷可用身份

3. **剧情包彻底分离**
   - 移除所有内置剧情（删除 `getFallbackDefaultPack` 函数）
   - 用户必须上传本地剧本或使用云端剧本
   - 剧本加载失败时显示友好提示而非使用内置备用剧本

### Bug修复

1. **超时扣血机制优化**
   - 实现渐进式扣血：超时后每秒扣血逐渐加速
   - 第1-5秒：每秒扣1点
   - 第6-10秒：每秒扣2点
   - 第11-15秒：每秒扣3点
   - 第16秒起：每秒扣5点（持续加速至致命）
   - 倒计时UI显示当前扣血速度

2. **生命值耗尽立即结束**
   - 修复了之前生命值归零后仍可继续答题的问题
   - 生命值归零时立即触发游戏结束
   - 不再使用延迟结束（移除 `setTimeout` 包裹）

1. 学识点作为唯一货币
   - 答题正确获得学识点
