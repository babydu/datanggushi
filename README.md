# 历史生存模拟器

一个基于 Web 的历史知识学习游戏，玩家可以在历史长河中"生存"，通过回答历史问题来体验不同朝代的人生历程。

## 功能特点

- 多卷解锁机制：强制从第一卷开始，通关解锁下卷
- 属性系统：生命值、求救次数、负面状态等
- 限时答题：每道题有时间限制，超时扣血
- 随机选项池：每次答题选项随机抽取
- 多结局判定：根据属性和答题情况评定成绩
- 完整复盘：游戏结束后可查看完整闯关记录
- 知识点详解：回顾游戏中的历史知识点
- 本地排行榜：记录个人最佳成绩

## 游戏剧本

### 内置剧本

| 剧本 | 朝代 | 卷数 | 关卡数 |
|------|------|------|--------|
| 宋朝历史生存剧本 | 北宋 | 2卷 | 每卷6+关卡 |

### 剧本内容

- **东京日迹**：体验仁宗年间开封府的市井生活，可选身份包括商铺掌柜、寒门学子
- **盛世危言**：在边疆危机和朝政变革中做出抉择，可选身份包括边疆将领、市井郎中

## 技术架构

- 纯前端实现，无需后端（可选 Python 服务器提供剧本 API）
- 框架与剧情分离的 JSON 剧本格式
- LocalStorage 实现存档、排行榜、知识点存储
- 响应式设计，支持移动端

## 本地运行

### 方式一：直接打开

直接双击 `index.html` 在浏览器中打开（需要确保 `data/song_pack.json` 在同一目录下）

### 方式二：使用 Python 服务器（推荐）

```bash
cd datanggushi
python3 server.py
```

然后访问 http://localhost:8080

### 方式三：使用静态文件服务器

```bash
cd datanggushi
python3 -m http.server 8080
```

访问 http://localhost:8080

```json

    "author": "作者",
    "description": "剧本描述"
  },
  "attributeConfig": { ... },
  "timeLimitConfig": { ... },
  "scoreConfig": { ... },
  "specialEvents": [ ... ],
  "volumeList": [
    {
      "id": "卷ID",
      "name": "卷名称",
      "identityList": [
        {
          "id": "身份ID",
          "name": "身份名称",
          "levelList": [
            {
              "level": 1,
              "story": "剧情描述",
              "optionPool": [ ... ]
            }
          ]
        }
      ]
    }
  ]
}
```

## 开发相关

### 添加新剧本

1. 在 `data/` 目录下创建新的剧本 JSON 文件
2. 参考 `data/song_pack.json` 的格式编写剧本
3. 确保所有必填字段完整
4. 可在服务器端 `server.py` 中添加新的 API 端点

### 修改默认剧本

修改 `game.js` 中的 `DEFAULT_PACK_URL` 常量指向新的剧本文件
